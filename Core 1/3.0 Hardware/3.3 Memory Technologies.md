## Memory that checks itself
- **Used on critical computer systems**
	- VM servers, database servers, any server
- [**Parity memory**](<##Parity>)
	- Adds an additional parity bit
	- Won't always detect an error
	- Can't correct an error
	- It can still tell you that a memory error has occurred
- [**Error Correction Code (ECC)**](<##ECC>)
	- Detects errors and corrects on the fly
	- Not all systems use ECC
	- It looks the same as non-ECC memory

## Parity
> A **parity bit** is a simple form of error detection used in digital communications, computing, and data storage. <mark>It is an extra bit added to a binary code to ensure the accuracy of data transmission or storage.</mark> The value of the parity bit is determined based on the number of ones (or zeros) in the data being transmitted. Its purpose is to enable the receiver to detect errors that may have occurred during transmission. [^1]
- Parity bit = a ninth bit
	- Also technically a non-ECC RAM
- **Even parity**
	- The parity that makes an even number of bits in the total number of a byte
- **Valid or error?**
	- Even parity byte check
	- ![](<./assets/3.3 Memory Technologies/file-20260310170356113.png>)[^2]
		- The parity bit will detect an error in the memory
			- If the 1s don't make up an even number / if the 1s make up an odd parity
		- The receiver will conclude that the error has occurred[^1]
			- The parity bit cannot specify what kind of error or how to resolve it.


## ECC memory
> **ECC**, or Error-Correcting Code, is <mark>a method used to detect and correct errors in data transmission or storage</mark>. It adds extra bits to the original data, creating redundancy that allows errors to be identified and corrected, ensuring data integrity. This redundancy allows the receiver to detect and correct errors that may occur due to noise, interference, or other factors. ECC is crucial in critical systems where accuracy is paramount, such as servers, networking equipment, and storage devices, ensuring reliable communication and data integrity. [^3]

- **ECC RAM (Error-Correcting Code RAM)** [^8]
	- Detects & corrects single-bit memory errors automatically
	- Ideal for environments where data accuracy & uptime are critical (servers, workstations, financial systems, medical devices)
	- Contains an extra chip (9 total) dedicated to error detection and correction
	- Slightly slower due to the additional error-checking process
	- More expensive, reflecting its reliability & specialized use

### How ECC Resolves Errors:
1. **Single-Bit Errors:** If one bit flips, ECC identifies exactly which one it is and flips it back to the correct state **on the fly**. The user never even knows it happened.
    
2. **Double-Bit Errors:** If two bits flip, ECC acts like Parity—it detects the error but cannot fix it, resulting in a system crash to protect data integrity.

## ECC vs. Non-ECC RAM

### ECC vs Parity - Comparison

|**Feature**|**Non-Parity (Standard)**|**Parity (Legacy)**|**ECC (Modern Server/Workstation)**|
|---|---|---|---|
|**Detection**|None|Yes|Yes|
|**Resolution**|None|**None (System Crash)**|**Automatic Correction** (Single-bit)|
|**Cost**|Cheapest|Moderate|Most Expensive|
|**Typical Use**|Home/Gaming PCs|Old Workstations|Servers / Financial Systems|
- If an exam question mentions a **Server** or a **Workstation** that needs to stay up 24/7 without crashing, they are looking for **ECC** as the answer. If the question asks about a simple error detection method that results in a system halt, that's **Parity**.

### Non-ECC RAM [^8]
- Standard memory used in most consumer desktops & laptops
- Does not perform error correction, but offers faster speeds & lower costs
- Perfect for gaming, office work, & general home use

### How to Identify the Difference (ECC RAM vs. Non-ECC RAM) [^8]
- **ECC RAM**
	- 9 chips (8 data + 1 ECC)
	- Labeled "ECC" or "registered"
- **Non-ECC RAM**
	- 8 chips 
	- Lacks ECC labeling
- **ECC Modules require compatible motherboards**
	- Non-ECC boards will not support them 

#### Exam Tip 
- Use ECC RAM for mission-critical systems where stability matters
- Use Non-ECC RAM for everyday computing to save cost and improve speed


| Feature            | ECC RAM               | Non-ECC RAM       |
| ------------------ | --------------------- | ----------------- |
| **Error Checking** | Yes (single-bit)      | No                |
| **Cost**           | Higher                | Lower             |
| **Performance**    | Slightly slower       | Faster            |
| **Use Case**       | Servers, workstations | Home PCs, laptops |

## Buffered vs. Unbuffered RAM
### Buffered (Registered) RAM [^7]
- Contains a buffer (register) between the DRAM and memory controller.
- Allows for more memory modules and greater total RAM capacity.
- Commonly used in enterprise-grade servers.
- Slight latency tradeoff due to the buffer.
- Used in servers & workstations, not standard desktops. [^8]

### Unbuffered RAM [^7]
- Sends signals directly from the memory controller to the DRAM.
- Faster and more responsive for general -purpose computing.
- Used in nearly all consumer desktops and laptops.

### Key Differences: Unbuffered vs. Buffered

|**Feature**|**Unbuffered (UDIMM)**|**Buffered/Registered (RDIMM)**|
|---|---|---|
|**Middleman**|None (Direct CPU access)|Yes (Register chip acts as intermediary)|
|**Stability**|Standard; sensitive to electrical load|High; designed for massive capacity|
|**Speed**|Slightly faster (no middleman delay)|Slightly slower (due to one extra clock cycle)|
|**Typical Use**|Desktops, Laptops|Servers, High-end Workstations|

## CPU to RAM throughput
- **There's a maximum throughput between RAM and CPU**
	- The memory bandwidth
- **Measured in MT/s**
	- Megatransfers/s
	- Million transfers per second
	- *e.g., 32 GB DDR5, 1x32GB, 5600MT/s*
		- 32-gigabyte DDR5 memory module
		- 1-by-32-gigabyte
		- Module supports 5600 million transfers per second
	- "DDR4-3200" = 3200 MT/s speed rating [^8]
	- "PC4-25600" = Same module expressed as total bandwidth (3200 x 8 bytes) [^8]
- **Faster is better**
	- Physics becomes the challenge
	- If we want to increase the speed of a system,
		- You would generally need to increase the data rate between the CPU and the memory

## Channel Configurations
- **Overview**
	- RAM can work alone or as part of a multi-stick configuration
	- Channel setup affects speed, bandwidth, and multitasking performance
- **Single Channel**
	- <mark>Uses one RAM stick</mark> operating independently
	- Lowest performance - limited memory bandwidth
	- Not ideal for gaming or multitasking
- **Dual Channel**
	- <mark>Two sticks</mark> work together to share the data load
	- Each handles half the data bus width, effectively doubling bandwidth
	- Most commonly set up in modern desktops & laptops
	- Provides excellent performance for everyday users & gamers
- **Triple Channel**
	- <mark>Utilizes three RAM sticks</mark> working in parallel
	- Found on certain older Intel motherboards
	- Rare today but may appear in legacy server systems
- **Quad Channel**
	- <mark>Uses four sticks of RAM</mark> simultaneously
	- Found in *high-end desktop (HEDT)* & server systems
	- Provides max bandwidth – great for video editing, 3D rendering, & scientific workloads
- **Important Setup Notes**
	- Match the correct number of sticks (2 for dual, 4 for quad, etc.)
	- Ensure identical size, type, & speed across all sticks
	- Install in the correct motherboard slots (check color coding / manual)
- **Tip**
	- Mixing different RAM sizes (e.g., 4GB + 8GB) can disable multi-channel mode
	- Always use matched pairs or sets for the best performance  [^8]

### Multi-channel memory
![](<./assets/3.3 Memory Technologies/file-20260310173546652.png>)[^4]
- **Difficult to increase the speed**
	- Push as much as possible across a single memory bus
	- We hit a maximum of what is humanly possible
	- If you're moving information from memory into a CPU, and then back to memory again,
		- You may find that you can max out the total throughput available on that particular pathway.
		- Challenges 
			- The CPU has a bit of idle time while waiting for the memory transfer to occur.
			- We could be performing other calculations while this happens.
			- The bandwidth between memory and CPU may not be enough to support the overall speed of the CPU.
![](<./assets/3.3 Memory Technologies/file-20260310174223037.png>)[^5]
- **Two independent memory buses**
	- Doubles the throughput
	- Two memory modules are faster than one
	- To increase these throughputs,
		- You may need to add an additional channel between the CPU and the memory.
		- This allows the CPU to communicate to multiple memory modules simultaneously
		- This helps increase the overall throughput of the system.
	- Having multichannel memory support could potentially double the total throughput between your memory and your CPU.
![](<./assets/3.3 Memory Technologies/file-20260310174735074.png>)[^6]
- **Dual-channel, triple-channel, or quad-channel**
	- Check your motherboard documentation
	- Increase the total throughput of that bandwidth by the total amount
- **Memory combinations should match**
	- Exact matches are best
	- The modules must be the same type of memory, ideally the same make and model
- **Memory module slots are often colored differently**
	- You may find your systems shipped with two 16GB memory modules rather than one single 32GB memory module.
	- Having two separate memory modules provides additional throughput between your CPU and your memory.

## Other RAM Specs
- **CAS Latency (CL)**
	- Measures how many clock cycles it takes for RAM to deliver the requested data
	- <mark>Lower CL = faster performance</mark>
	- Ex. CL16 is faster than CL19 (when comparing the same DDR generation)
	- Important for performance tuning & overclocking scenarios
- [Registered (Buffered) RAM](<## Buffered vs. Unbuffered RAM>)
- **XMP (Extreme Memory Profile)**
	- A feature that allows RAM to run beyond standard speeds safely
	- Predefined performance profiles stored in the RAM module
	- Activated through the BIOS / UEFI settings
	- Popular among gamers & enthusiasts for achieving maximum performance
